使い方

  1. 使いたいオプションにチェックを入れ、必要なら値を入力する
  2. 上の出力欄に生成された文字列が出るので「コピー」ボタンを押す
  3. Steam: ライブラリで Apex を右クリック → プロパティ → 起動オプション欄に貼り付け
    EA アプリ: ライブラリで Apex の「…」→ プロパティ → 詳細起動オプション欄に貼り付け
  4. すでに設定済みの文字列を読み込みたい場合は「▸ 既存の文字列から読み込む」を押して貼り付け

注意書き
保存はブラウザの localStorage を使用しています。同じブラウザで再度開けば設定は自動的に復元されます。
各オプションの説明・効果・推奨値は EA 公式ヘルプ・Valve Developer Wiki・PCGamingWiki・有志 Wiki / 解説記事に基づきます。Apex はアップデートで挙動が変わるオプションがあるため、出典記載のないオプションや「注意」付きのオプションは現バージョンで効くか各自で確認してください。

グラフィック系 ConVar (r_shadowrendertotexture / r_shadows / r_lod_switch_scale / mat_disable_bloom など) について

まず前提として、Apex の起動オプション欄は +<ConVar> <値> という書式を解釈する仕組みを持っており、チート保護されていない ConVar (本ツールが載せている +fps_max、+cl_showpos、+cl_showfps、+m_rawinput、+mat_queue_mode など) は 起動オプション経由でも問題なく機能します。
一方で、シャドウや LOD・テクスチャフィルタリング・ブルームなどを切り替える r_* / mat_* 系の 一部 ConVar を起動オプションの欄に +r_shadowrendertotexture 0 のように直接書いて使っている解説記事を見かけますが、これらは本来 autoexec.cfg に書く前提のコマンドで、起動オプションとして渡した場合に確実に効くという信頼できる検証情報は見つかっていません (Respawn 側でロード順や保護設定により実質ブロックされている疑い)。本ツールがこれらを「起動オプション」のチェック項目として載せていないのはこの理由です。
使いたい場合は <Apex インストール先>/cfg/autoexec.cfg にコマンドを記述し、本ツールの「上級者向け」にある +exec autoexec.cfg を有効にして読み込ませてください。
なお、EA は公式に「パフォーマンス調整は基本的にゲーム内オプションを使うのが望ましい」とコメントしており、autoexec の使用は競技シーン (ALGS など) では制限／非推奨となる場合があります。
